# 実装レビュー観点

Codex が実装を返した後、ClaudeCode は以下の観点でレビューを行います。

## レビュー観点（全8項目）

### 1. ユーザーの指示に忠実か
- ユーザーが明示した画面・目的・確認ポイントが反映されているか
- ユーザーが指定していない要素が勝手に追加されていないか

### 2. Codex 実装プロンプトに沿っているか
- `.codex-prompt.tmp.md` の `Implementation requirements` を満たしているか
- `Acceptance criteria` を全て満たしているか

### 3. 指示にない仕様追加がないか
- `Must not do` に記載した禁止事項が守られているか
- Edit scope 外のファイルが変更されていないか

### 4. 静的モックとして必要な表現が欠けていないか
- 顧客が画面をイメージできる最低限の要素（ラベル・ボタン・レイアウト）が揃っているか
- 仮テキスト・プレースホルダが適切に配置されているか

### 5. 画面・構成・テキストが不自然に膨らんでいないか
- 指示になかった画面・セクション・項目が追加されていないか
- テキスト量が過剰に多くなっていないか

### 6. 本番実装のような余計な機能が混ざっていないか
- 指示にない新規 JavaScript が追加されていないか
- インラインスクリプト（`<script>` タグ直書きロジック）が混入していないか
- 既存 JS（`auth.js` / `shared-layout.js` / `shared-mobile-layout.js` 等）の重複実装・置き換えが行われていないか
- API 通信・フォーム送信・外部サービス連携が実装されていないか
- 既存のモック JS 基盤（認証・セッション・レイアウト注入）は許容範囲のため、その存在だけで問題視しない

### 7. HTML/CSS の実装として過剰に複雑化していないか
- 静的モック用途に対して不必要なクラス設計・ファイル分割が行われていないか
- 外部ライブラリ・フレームワークが無断で追加されていないか

### 8. 次の再実装指示が必要か
- 上記いずれかで問題がある場合、修正内容を整理して再実装指示を出す

## レビュー結果の判断形式

| 判断 | 意味 |
|---|---|
| **OK** | ユーザー指示・プロンプト・品質の全観点で問題なし |
| **要修正** | 具体的な問題点を列挙し、再実装指示を作成する |
| **保留 / ユーザー確認推奨** | 仕様判断が必要な点があり、ClaudeCode だけでは決定できない |

## 報告形式

レビュー結果をユーザーに返す際は以下の構造で報告する：

```
## レビュー結果：[OK / 要修正 / 保留]

### 確認済み観点
- （問題なかった観点）

### 問題点（要修正の場合）
- （具体的な問題と修正内容）

### ユーザーへの確認事項（保留の場合）
- （確認してほしい点）
```
