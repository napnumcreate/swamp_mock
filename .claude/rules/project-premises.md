# プロジェクト前提

このファイルは本モック制作の前提条件を定めます。
Codex プロンプト作成・レビュー時に必ず参照してください。

## 前提1：システム概要

**ホストクラブの店内業務システム**のモックです。
実際の店舗スタッフが業務で使用することを想定しています。

### 画面ごとの補足

- **店舗状況**：どのテーブルに誰が座っているかを把握・管理する画面
- **注文**：テーブルごとのドリンク・フードの注文を入力・管理する画面
- **売上**：ホスト自身の売上・実績を確認する画面
- **顧客**：来店顧客の情報を参照する画面（モバイル）
- **設定**：ホスト・内勤それぞれのアプリ設定画面
- **給料計算**：内勤スタッフとホストは別計算。それぞれの給与を算出・確認する画面
- **勤怠管理**：ホストのシフトが一覧で確認できる画面（PC閲覧）
- **アナリティクス**：ホストごとの売上・実績などを可視化する画面
- **顧客管理**：来店顧客の情報を管理する画面（PC）
- **ホスト一覧**：在籍ホストの一覧・プロフィール管理画面
- **データ管理**：マスタデータ・設定データの管理画面

## 前提3：デザイン方針

- **白基調で高級感のあるデザイン**
- 理由：店内が暗いため、スタッフがスマホ・PCを見たとき視認性を確保するため
- トーン：クリーン・エレガント・高級感。過度な装飾はせず、情報を整理して見せる

### デザイン適用のポイント

- 背景は白または明るいグレー系
- フォントは読みやすいサンセリフ系
- スマホ画面（mobile）はタップ操作しやすいボタンサイズ・余白を確保
- PC画面（docs/pages）はテーブル・カード型の情報レイアウトを基本とする

### アクセントカラー体系（確定仕様）

| 画面区分 | アクセントカラー | 用途 |
|---|---|---|
| PC（`docs/pages/`） | ゴールド `#C9A84C` | 管理者向け画面全般 |
| モバイル・ホスト向け | パープル `#7C3AED` | `.mobile-shell--host` で適用 |
| モバイル・内勤向け | ブルー `#2563EB` | `.mobile-shell--staff` で適用 |

アクセントカラーは CSS 変数（`--color-accent` 等）で管理されており、`.mobile-shell--host` / `.mobile-shell--staff` クラスでロール別に上書きされる。
